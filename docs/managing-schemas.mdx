# Managing Schemas

### **1. Creating a Schema**

- **Importing Schema**
  - Start by importing the desired schema (e.g., BookSchema).
  - The schema can be edited before saving.
- **Saving as Draft**
  - Initially, the schema is saved as a draft version.
  - Changes to the schema (e.g., renaming it to CompanySchema) will override the existing draft.
  - You can either publish or discard the draft.

### **2. Publishing a Schema**

- Once the schema is finalized, publish it as the **initial version**.
- After publishing
  - The schema enters the **Published State**.
  - A side panel displays
    - Schema version.
    - Comments.
    - Publication timestamp.

### **3. Managing Versions**

- **Previewing and Deprecating**
  - Use the **Preview** option to view the schema details.
  - Use the **Deprecate** option to mark a version as deprecated. This is a soft delete, meaning the version is not removed but marked as deprecated.
  - Note: The latest version cannot be deprecate&#x64;**.**

<figure>
  <img src="/img/assets/SR-Preview-and-Depricate.gif" alt=""></img>
  <figcaption></figcaption>
</figure>

- **Editing and Creating New Versions**
  - Editing a published schema creates a new draft.
  - When published, this draft becomes a new version.

<figure>
  <img src="/img/assets/SR-Edit.gif" alt=""></img>
  <figcaption></figcaption>

- **Version Control**
  - You can duplicate a previous version for reuse or modification.
  - The application alerts you if you're not using the latest version of the schema.

### **4. Updating Schema in Applications**

- Go to the **Schema Registry** and search for the schema (e.g., BookSchema).
- Applications can import specific versions of a schema
  - If an older version is in use, alerts notify you about newer versions available.
  - You can update to the latest version, but once updated, reverting to a deprecated version is not allowed.

<figure>
  <img src="/img/assets/SR-update.gif" alt=""></img>
  <figcaption></figcaption>
</figure>

---

### **5. Promoting a Schema**

- Create a copy of the application schema.
- Use the **Promote** option to
  - Add the schema to a collection.
  - Validate it for uniqueness in both collections and applications.
- After promotion
  - The schema becomes a **global schema**.
  - It is added to the appropriate collection.
