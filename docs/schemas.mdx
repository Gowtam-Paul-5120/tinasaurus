# Schemas

A **schema** is like a blueprint for your data. It defines how your data is organized, what type of data it contains, and the rules it must follow. In our application, schemas ensure that your workflows and operations run smoothly and consistently.

### **1. Creating a Schema**

Want to create a schema from scratch? Follow these steps

1. **Go to the Schema Section**: Navigate to the **Schemas** section in the application.
2. **Click Create Schema**: Start a new schema by clicking **Add a Schema**.
3. **Name Your Schema**
   - Use descriptive names that represent your data.
   - **Important**: Schema names cannot have spaces and start with lowerCase. Use underscores (`_`) or UpperCase instead (e.g., `CustomerDataSchema)`.
4. **Design the Schema**
   - Add fields like `name`, `age`, or `email`.
   - Choose the data type for each field, such as `string`, `number`, or `boolean`.
   - Mark fields as required if theyâ€™re essential.
5. **Save Your Schema**: Once youâ€™ve defined the structure, click **Save** to finalize the schema.

### **2. Importing a Schema**

**Where Can You Import a Schema?**

You can import a schema into your workflow from three main locations

1. **Schema Folder**
   - Navigate to the **Schemas** folder and import your schema directly into the folder for centralized management.
   - Use this method to organize schemas across workflows.

1. **Input Tab of Activities**
   - While configuring the input fields of an activity, import a schema to define the expected data structure.

1. **Output Tab of Activities**
   - While configuring the output fields of an activity, import a schema to specify the structure of data being passed forward.

**Steps to Import a Schema**

1. **Choose Import**
   - Go to one of the locations above (Schema Folder, Input Tab, or Output Tab).
   - Click **Import Schema**.
2. **Select the Source**
   - **From JSON File**: Upload a JSON file containing your schema structure.
   - **From Schema Registry**: Search for a pre-existing schema in the centralized Schema Registry.

<figure>
  <img src="/img/assets/Schema-SchemaRegistry.gif" alt=""></img>
  <figcaption></figcaption>

1. **Review and Save**
   - Review the schema for accuracy, make adjustments if needed, and click **Save**.

### **3. Deleting a Schema**

To delete an unwanted schema

1. **Locate the Schema**: Open the **Schemas** section and find the schema you want to remove.
2. **Click Delete**: Right click on the Schema and select **Delete** Option.
3. **Confirm Deletion**: Confirm your action in the popup to permanently delete the schema.

### **4. Using a Schema**

Schemas can be directly applied to activities to define the input or output data structure

1. **Open the Activity Configuration**: Go to the activity where you want to use a schema.
2. **Attach the Schema**
   - For input data: Use the **Input** tab.
   - For output data: Use the **Output** tab.
3. **Map Data Fields**: Align the fields in the schema with the activityâ€™s data requirements.
4. **Save Configuration**: Save your changes to apply the schema to the activity.

#### **Best Practices**

1. **Use Clear Names**: Schema names should reflect their purpose (e.g., `OrderDetails` or `CustomerInfo`).
2. **Avoid Spaces**: Stick to underscores (`_`) or UpperCase for naming conventions.
3. **Validate Imports**: Double-check imported schemas to ensure compatibility with your workflow.
4. **Organize with Schema Folders**: Keep your schemas organized in folders for easier management.
