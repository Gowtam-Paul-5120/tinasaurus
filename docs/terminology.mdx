# Terminology

| Terms                                                                                                | Description                                                                           |
| ---------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| [Workspace](../working-with-workspaces/)                                                             | A collaborative environment for organizing and managing applications and workflows.   |
| [Application](../working-with-applications/)                                                         | A container within a workspace that holds workflows, resources, and schemas.          |
| [Workflow](../working-with-workflows/)                                                               | A sequence of steps designed to complete a process or achieve a business goal.        |
| [Activity](../working-with-activities/)                                                              | A fundamental task or operation within a workflow, like data entry or API requests.   |
| [Variable](../working-with-applications/variables)                                                   | Configurable parameters used for dynamic data storage and workflow customization.     |
| [Trigger](../working-with-workflows/#id-1.-create-a-workflow)                                        | An event or condition that initiates a workflow, such as time-based or user actions.  |
| [Schema](../working-with-applications/schemas)                                                       | Defines the data structure, including fields and data types, used in workflows.       |
| [Resource](../working-with-applications/resources)                                                   | External configurations or connections (e.g., Auth, ABS) used in workflows.           |
| [Subflow](../working-with-workflows/creating-and-using-a-subflow)                                    | A reusable sequence of steps within a larger workflow to modularize complex tasks.    |
| [Mapper](../working-with-workflows/managing-workflows/mapping/)                                      | Transforms and maps data between different formats or structures within workflows.    |
| [Mapper Function](../working-with-workflows/managing-workflows/mapping/allof-and-anyof)              | Performs data transformations using built-in functions for complex data mappings.     |
| [Webhook](../working-with-activities/#webhook)                                                       | Enables real-time communication by sending HTTP requests when specific events occur.  |
| [Error Handling](../working-with-activities/#generateerror)                                          | Manages and responds to errors within workflows for robust and resilient execution.   |
| [Lookup Table](../working-with-workspaces/managing-workspace-level-entities/managing-lookup-values/) | A key-value data structure for efficient data retrieval and mapping in workflows.     |
| [Transition Line](../working-with-workflows/managing-workflows/transition-lines)                     | Connects activities in a workflow, with optional conditions to control flow logic.    |
| [API Manager](../api-manager/)                                                                       | Manages API collections, access profiles, and secure API tokens for integrations.     |
| [Secret Manager](../working-with-workspaces/managing-workspace-level-entities/managing-secrets/)     | Secures sensitive information like API keys, supporting environment-specific storage. |
| [Publish](../working-with-workflows/managing-workflows/publishing-workflows)                         | Publish a specific version of the workflow, making it ready for deployment.           |
| [Deploy](../working-with-workflows/managing-workflows/publishing-workflows)                          | Moves the published version to the Development environment for execution.             |
| [Promote](../working-with-workflows/managing-workflows/publishing-workflows)                         | Advances the deployed version to Production environments with approvals.              |
| [Test](../working-with-workflows/managing-workflows/testing-a-workflow)                              | Verifies the workflow functionality by simulating input data and checking output.     |
| [Schema Registry](../working-with-the-schema-registry/)                                              | A centralized location for managing, versioning, and sharing schema definitions.      |
